Appendix A — If Josh disappears tomorrow

This page is written for someone else — a competent developer you've hired, if Josh is ever unreachable and you need help. It's deliberately one page: what exists, where it lives, how to restore it from nothing, and how to run it day to day. Nothing below depends on Josh being available, or even being asked.

Everything named on this page is already yours — in your accounts, under your control, before this page is ever needed. A developer reading this needs no introduction from Josh and no access he hasn't already been given by you, in Track P of this guide.

Where to get in

Server access needs no separate software: DigitalOcean's own website has a "Console" button on your droplet's page, opening a full terminal in your browser. The one script that built the whole thing lives at deploy/standalone/setup.sh inside the wixy-engine repository — safe to read, and safe to run again from scratch if ever needed.

How to restore, from nothing but a backup

  1. Create a new DigitalOcean droplet — same choices as Track P, chapter 3 (Marketplace "Docker on Ubuntu," Basic Regular, 2 GB / 1 vCPU, London).
  2. Run the same one-line setup command as Track P, chapter 6, answering with the same values (all in the password manager). This gives you a clean, working, empty website.
  3. Clone the latest snapshot from ca-state-backup's snapshot branch, and copy its contents into the running website's data volume (docker compose cp, or an equivalent mount-and-copy — it's a plain file tree, nothing proprietary or exotic).
  4. Restart the wixy container so it picks up the restored state.
  5. Point the domain at the new droplet's tunnel — same mechanism as Track P, chapter 8.

How to run it, day to day

  • Publishing a change: the admin area's Publish button, at /admin on your domain (behind Cloudflare Access).
  • Engine updates: Settings → Engine card → "Get engine updates," with a one-tap "Undo last update" if anything ever goes wrong.
  • Logs and troubleshooting: logs.sh, in deploy/standalone/, run from the droplet's own console.
  • The entire running stack is defined in one file — deploy/standalone/docker-compose.yml, inside the engine repository. Nothing about how this website runs is hidden anywhere else.
